# Oracle Corporation（ORCL）｜研究報告

**產業：** 企業軟體 / 雲端基礎建設
**市場：** US（NYSE）
**主題：** AI 資料中心基礎建設浪潮的直接受益者——OCI 高速擴張重塑雲端競爭格局
**研究日期：** 2026-03-10
**研究觸發：** 今日（3/10）盤後發布 Q3 FY2026 財報，多方分析師聚焦其 AI 雲端資料中心支出成長是否延續；若超預期則為 AI 基礎建設敘事的重要正面確認，若未達預期則具有「煤礦中的金絲雀」效應（CNBC 語）。是本週最重要的 AI 資本支出驗證節點，值得建立 thesis 追蹤

---

### 核心業務與商業模式

Oracle 的業務結構是「舊世界穩定、新世界加速」的雙軌組合，三大支柱並行：

**1. 資料庫與技術授權（最強護城河）**
- **Oracle Database**：全球市佔率第一的關係型資料庫，覆蓋全球 98% 的 Fortune 500 企業
- 客戶每年繳交原始授權費 22% 的年費支援（Support Revenue），毛利率接近 100%，幾乎不可能取消
- **Autonomous Database**：AI 驅動自動調參、修補、擴縮容的自治版本，是雲端遷移橋梁
- **MySQL HeatWave**：開源資料庫的雲端托管版，整合 OLAP+ML，主攻 AWS Aurora

**2. 雲端基礎建設（OCI — 最核心成長引擎）**
- 全球第四大超大型雲端供應商，GPU 算力定價比 AWS/Azure **低 30-50%**
- AI GPU 叢集（NVIDIA H100/H200、AMD MI300）為核心產品，AI 訓練需求直接受益
- **Stargate 計畫**：Oracle + SoftBank + OpenAI 三方合資，500 億 USD AI 資料中心建設，Oracle 是基礎建設核心供應商
- **反直覺的多雲策略**：將 Oracle Database 部署到 Azure、Google Cloud、AWS，客戶無需離開競品雲也能用 Oracle，反而深化資料庫依賴

**3. 企業應用 SaaS**
- **Oracle Fusion ERP/HCM/SCM**：中大型企業財務、人資、供應鏈一體化雲端 SaaS
- **NetSuite**：40,000+ 中型企業客戶、20%+ 年成長、100 國覆蓋，SMB ERP 的全球龍頭
- **Oracle Health（Cerner）**：2022 年 283 億 USD 收購，覆蓋美國近 50% 醫院的電子病歷系統

**收入結構：**

| 類型 | 占比 | 特性 |
|------|------|------|
| 雲端服務（OCI + SaaS） | ~45% | YoY +25-30%，未來主力 |
| 授權支援（年費） | ~40% | ~95% 毛利率，高黏性、緩降 |
| 授權 + 硬體 | ~15% | 一次性，逐漸轉訂閱 |

### 競爭護城河與市場地位

**護城河一：資料庫遷移的天文成本（最強壁壘）**
企業核心系統（銀行、ERP、製造 MES）的 Oracle Database 替換成本極高——需重寫數十萬行 PL/SQL 程式碼、6-24 個月資料遷移驗證。結果：即使有 AWS/Azure，企業仍乖乖繳交 22% 年費。

**護城河二：多雲互通——以毒攻毒**
Oracle Database@Azure / Google / AWS 讓 Oracle 將競爭對手的雲端基礎建設變成自己的分銷管道。客戶越依賴公有雲，反而越無法脫離 Oracle 資料庫。

**護城河三：OCI 的性價比差異化**
GPU 算力比 AWS 便宜 30-50%，網路出站費用接近免費（AWS 的 1/10）。AI 訓練公司多採多雲策略，OCI 因性價比成為首選的補充或主要供應商。

**護城河四：政府與醫療的結構性壁壘**
FedRAMP High + DoD IL5 認證，聯邦市場僅 Oracle 與 AWS GovCloud 二選一。Cerner EHR 替換是醫院最痛苦的 IT 項目，通常需要 2-5 年。

**市場地位：**

| 業務線 | 地位 | 主要競爭對手 |
|--------|------|------------|
| 企業關係型資料庫 | **全球第一**（~30% 市佔） | Microsoft SQL Server、PostgreSQL |
| 企業 ERP（大型） | **全球第二**（~10%） | SAP（#1，~25%）|
| 雲端 ERP（SMB） | **NetSuite 全球龍頭** | Microsoft Dynamics、Sage |
| 電子病歷（EHR）美國 | **全球第二**（~25%） | Epic（#1，~35%）|
| 雲端基礎建設 | **第四**（~2-3%） | AWS、Azure、Google Cloud |

### 成長驅動力

1. **OCI GPU 叢集 AI 需求爆發**：Cloud RPO +80% 已驗證結構性需求，Stargate 計畫讓 Oracle 成為「美國 AI 國家隊」基礎建設商。OCI 雲端收入有望從 ~200 億 USD 規模加速至 400 億 USD+。

2. **多雲資料庫策略的收入升級**：在 Azure/Google/AWS 運行 Oracle Database 的客戶，付費計入 Oracle 雲端收入，傳統授權客戶自然轉化為雲端訂閱，同一客戶年費提升 30-50%。

3. **NetSuite 複利滾動**：全球中型企業 ERP 現代化率仍低，留存率 >95%，20%+ 年成長可望持續至 2028 年以上。

4. **Oracle Health AI 轉型**：V8 平台嵌入 AI 臨床決策支援，若整合成功，每床位 ARPU 大幅提升，TAM 超過 2,000 億 USD（美國醫療 IT）。

5. **政府 AI 雲端優先供應商**：Stargate 政治意義讓 Oracle 卡位聯邦/國防長期合約（5-10 年週期），可預見性極高。

### 主要風險

**1. 大規模 CapEx 致 FCF 轉負（最大近期風險）**
FY2026 CapEx 預計翻倍至 180 億 USD+，在收入認列時間差下 FCF 短期轉負。Oracle 淨債務超過 800 億 USD，財務槓桿高。Barclays 近期下調目標價的核心依據即是此點：CapEx 大週期的財務負擔 + 毛利率短期壓縮。若 AI 需求未如預期兌現，債務壓力將顯著放大。

**2. OCI 能否突破「補充供應商」定位的競爭不確定性**
AWS、Azure、Google Cloud 生態系遠比 OCI 完整，若主要競爭對手跟進 GPU 降價，OCI 差異化削弱。AI 訓練需求若在 2026-2027 年出現週期性修正（算力過剩），GPU 合約新簽速度放緩，RPO 成長趨勢將逆轉，直接衝擊 thesis 核心。

**3. Cerner 整合落後與 Oracle Health 結構性挑戰**
283 億 USD 收購後整合進度慢於預期，V8 平台遷移複雜、監管限制多，客戶向 Epic 流失的案例仍持續出現。若整合失敗，潛在減值風險超過 200 億 USD，且拖累整體毛利率達 2-3 個百分點。

### 買入邏輯初步評估

> **Oracle 是 AI 基礎建設時代「舊世界護城河 + 新世界加速」的罕見組合。**
>
> 資料庫年費（~95% 毛利率）提供抗跌底蘊，OCI 的 Cloud RPO YoY +80% 提供高成長敘事。Forward P/E 19.25x 相對此成長組合具吸引力，分析師目標均價 253 USD 隱含 **67% 上行空間**。
>
> **Q3 FY2026 財報（2026-03-10）已完成 thesis 第一個驗證**：雲端成長 +44%（遠超 ≥25% 閾值）、OCI RPO 積壓持續擴大、管理層上修 FY2027 指引、股價盤後突破 160 USD（+8%）。所有進場條件已達標。
