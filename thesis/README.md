# thesis/ — 個股買入背景資料庫

每支持倉或觀察清單個股，對應一個 Markdown 檔案。
每日報告生成時，Claude CLI 會自動讀取相關檔案，驗證原始買入邏輯是否仍然有效。

## 檔案命名規則

`{TICKER}.md`，ticker 使用大寫，台股用數字代號：

```
thesis/
  3030.md     ← 德律科技
  CRWD.md     ← CrowdStrike
  NET.md      ← Cloudflare
  ALAB.md     ← Astera Labs
```

## 建議格式

```markdown
# {公司名}（{TICKER}）｜買入依據背景

**產業：** ...
**市場：** ...
**最後更新：** YYYY-MM-DD

---

### 核心業務
...

### 成長引擎 / 催化劑
...

### 財務試算
| 指標 | 數值 |
|------|------|
| ... | ... |

### 主要風險
- ...

### 買入邏輯總結
> 一段話總結為何買這支股票
```

## 更新時機

- 財報發布後（更新 EPS、指引數字）
- 重大新聞發生後（業務邏輯改變）
- 定期季度回顧（每季更新一次）
