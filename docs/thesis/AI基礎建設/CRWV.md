### 核心業務與商業模式

#### 公司背景：從挖礦到 AI 算力基礎設施龍頭

CoreWeave 由三位前對沖基金交易員於 2017 年創立，起初名為 **Magnetic Mining**，主業加密挖礦。2019 年洞察到 NVIDIA GPU 的真正價值在 AI/ML，迅速轉型為雲端算力租用服務——這段歷史使 CoreWeave 比任何雲服務商都更早在「大規模 GPU 部署與管理」上積累實戰經驗。

#### 賣什麼：AI 優先的 GPU 算力雲

**核心服務三層架構：**

| 服務層 | 產品 | 特徵 |
|--------|------|------|
| 算力層 | NVIDIA HGX H100/H200/B200/B300 叢集 | 千卡以上規模，InfiniBand 超低延遲互聯 |
| 容器編排層 | CoreWeave Kubernetes Service（CKS） | 針對 AI 工作負載優化的 K8s 管理平台 |
| 推論優化層 | NVIDIA Dynamo 1.0 協同整合 | 支援 Agentic AI 多步驟推論工作流程（GTC 2026 發布） |

**最關鍵的技術差異化：**

1. **InfiniBand 全網路架構**：多 GPU 訓練節點間通訊延遲降低至 AWS/GCP 的 1/5 以下，對大型 Transformer 模型訓練效率有決定性影響
2. **液冷高密度部署**：GPU 密度顯著高於傳統資料中心
3. **裸機存取（Bare Metal）**：AI 研究者可完全控制底層硬體，非被雲抽象層鎖死

**收入結構：**

| 收入型態 | 佔比（估） | 特徵 |
|---------|-----------|------|
| 長期預留合約 | ～80% | 1-5 年 take-or-pay，客戶預付或鎖定用量 |
| 按需算力 | ～15% | 短期彈性，溢價計費 |
| 管理服務 / 其他 | ～5% | Kubernetes 管理、技術諮詢 |

**客戶輪廓：**

| 客戶 | 合約規模 |
|------|---------|
| **Microsoft** | 2024 年占營收 ~62%，$10B+ 多年合約 |

| **Cohere** | 企業 LLM 訓練與推論 |

### 競爭護城河與市場地位

**5 道護城河：**

1. **Nvidia 戰略同盟**（決定性）：優先 GPU 供貨 + 聯合研發 Dynamo 1.0 + Rubin NVL72 首批部署夥伴 + 客戶引薦通路。Nvidia 先後直投 CoreWeave 和 Nebius，確立「Nvidia 認可的 Neocloud 雙核心」格局
2. **AI 工作負載技術深度**：InfiniBand 全網路架構、7 年 GPU 大規模管理經驗，後進者難以在短期複製相同規模
3. **長期合約鎖定**：take-or-pay 結構 + ABS 抵押融資飛輪（合約→融資→擴充 GPU→新合約），客戶切換成本極高
4. **規模壁壘**：~250,000 張 NVIDIA GPU，是多數 Neocloud 競爭者的 10 倍以上
5. **先發資料中心布局**：電力許可、冷卻改造、網路互聯均需 12-24 個月建設週期，地理位置優勢難以快速複製

**競爭格局：**

| 競爭者 | 類型 | 相對定位 |
|--------|------|---------|
| AWS / Azure / GCP | Hyperscaler | 規模更大但通用化，CoreWeave 是其「算力溢出閥」——互補而非純競爭 |
| CoreWeave（CRWV） | 美國最大獨立 GPU 雲 | AI 專業深度最強、Nvidia 直接背書 |
| Nebius（NBIS） | 歐洲最大獨立 GPU 雲 | 地理分工，歐洲市場主導 |
| Lambda Labs / Crusoe | 小型 Neocloud | 規模差距 10x 以上 |

### 成長驅動力

1. **Agentic AI 推論需求爆炸（最重要，2026-2027）**：每個 AI Agent 執行一個任務需數十至數百次推論呼叫，算力消耗倍增。CoreWeave + Nvidia Dynamo 1.0 針對此場景深度優化，先發優勢顯著

2. **Rubin NVL72 首批部署（2026 下半年）**：每代新 GPU 架構效能跳升 3-4 倍，首批部署者定價權最強、合約期限最長，是收入加速的重要觸媒

3. **客戶集中度分散**：Microsoft 62% 的高集中度反而意味著巨大的拓展空間，每新增一個大型客戶都能直接改善收入結構與風險輪廓

4. **歐洲 / 亞太地理擴張**：歐洲 AI 算力需求受 GDPR 數據主權推動，偏好非 Hyperscaler 的獨立 GPU 雲，CoreWeave 已布局歐洲資料中心

5. **軟體層貨幣化**：CKS 和 Dynamo 整合服務滲透率提升，將使每位客戶 ARPU 增加 20-30%，推動毛利率向 SaaS 水準靠攏

6. **企業 AI 全面部署潮（2026-2027）**：Fortune 500 從 AI 試點進入規模部署，長期合約模式契合企業 IT 預算規劃習慣

### 主要風險

**風險 1：客戶高度集中（Microsoft ~62%）【最大結構性風險】**
Microsoft Azure 正自建 Maia AI 晶片並擴充 GPU 算力，若 Microsoft 在合約到期後減少依賴，對 CoreWeave 營收衝擊巨大。反制論點：現有合約通常為 3-5 年期，即使開始分散，短期影響有限。**合約到期時程 vs Microsoft 自建進度是核心追蹤指標。**

**風險 2：資本結構脆弱——高槓桿雙刃劍【財務風險】**
合約抵押融資策略使有息負債估計達 $7B+，若 GPU 需求增速放緩、利率環境惡化、或客戶違約，高槓桿將快速轉化為流動性壓力。CoreWeave 毛利為正但自由現金流（扣 Capex 後）仍為負值。

**風險 3：Hyperscaler 自建加速與 GPU 稀缺性下降【競爭與技術風險】**
三重壓力：AWS/Google/Microsoft 加速自建 AI 晶片（降低 Nvidia GPU 依賴）+ Nvidia 自身擴張 DGX Cloud + GPU 供應鏈改善後稀缺性紅利消退。CoreWeave「優先取得最新 GPU」的護城河可能隨時間稀釋。
